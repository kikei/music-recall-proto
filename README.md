# 音楽想起エンジン (プロトタイプ)

音楽を「保存」するのではなく、後日ふたたび **想起** される可能性を作る個人用
ツール。聴いた印象を LLM との対話で言葉にし、小さな「再会カード」として残す。
後日、別の曲の印象や曖昧な言葉をきっかけに、過去のカードが浮上する。

コンセプトの詳細は [CONCEPT.md](./CONCEPT.md) を参照。

## できること (プロトタイプの範囲)

- **聴取セッション**: 対象 (曲・アルバム・ライブ盤など) とアーティストを入れて
  開始 → 気になった瞬間に短い
  言葉を投げると、音楽ソムリエ役の LLM が web も調べつつ言語化を手伝う →
  「再会カードにする」で 1 枚へ圧縮。
- **想起**: 今のきっかけ (印象や曖昧な言葉) を入れて「想起」を叩くと、意味検索
  で関連カードを少数だけ浮上させ、**なぜ今それが想起されたか** を説明する。
- **カード一覧 / 追記**: 溜まったカードを見る。再会して印象が深まったら追記。

外部サービス連携はなし。曲名・アーティストも含めすべて手入力。

## セットアップ

```sh
npm install
cp .env.example .env   # OPENAI_API_KEY を記入
npm run dev
```

`npm run dev` でバックエンド (Hono, :8787) とフロント (Vite, :5173) が同時に
起動する。ブラウザで <http://localhost:5173> を開く。

## 環境変数 (.env)

| 変数 | 既定 | 説明 |
| --- | --- | --- |
| `OPENAI_API_KEY` | (必須) | OpenAI API キー |
| `OPENAI_MODEL` | `gpt-5.5` | 対話・圧縮・想起理由 (web 検索対応) |
| `OPENAI_EMBED_MODEL` | `text-embedding-3-small` | 想起の意味検索 |
| `PORT` | `8787` | バックエンドのポート |

## データ

SQLite (`music-recall.sqlite`) にローカル保存。中心データ単位は「曲」ではなく
「聴取体験 (セッション)」。サンプルデータは同梱しない。まず自分で数枚カードを
作ってから想起を試す。

## 構成

- `server/` — Hono API。`db/` データ層、`llm/` OpenAI 連携、`cards/` 想起と
  カード生成、`routes/` エンドポイント。
- `src/` — React フロント。`screens/` が 3 画面、`api/` がクライアント。
